Bunnyland 3D

Out-of-tree 3D clients

Play and inspect Bunnyland worlds in 3D.

This container adds Three.js clients to a Bunnyland deployment while keeping the 3D plugin and web code outside the base server and web repositories.

Open 3D Player Open 3D Admin

Choose the surface that matches the job:

  • 3D Player follows a claimed character, keeps fog of war, shows actions, queues, portraits, stats, inventory, exits, and scene images.
  • 3D Admin renders the full world overview, room layout, server-provided 3D components, selectable room contents, and canvas captures.

Clients

Open a 3D view.

Both clients use the same Bunnyland API path as the base web container and accept a server query parameter for alternate servers.

3D Player

A character-scale third-person client for claimed characters, with local WASD exploration, animated low-poly avatars, biome rooms, spatial targets, confirmed exit travel, and a collapsible gameplay HUD.

Best for players: action search, target selection, command queue management, vitals, portrait, inventory, exits, and the scene image gallery.
Open Player

3D Admin

A full-world 3D inspector for room layouts, server-enriched 3D components, selectable entities, room contents, camera controls, and downloadable canvas captures.

Best for admins: checking world geometry, component projections, generated room spacing, movement paths, collision setup, and rendering regressions.
Open Admin
